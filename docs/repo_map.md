# Repository Map

## Core control files

* `README.md` for human usage guidance.
* `CLAUDE.md` for AI operating rules.
* `tasks/project_stage.md` for file activation and stage transitions.
* `project_brief.md` for product definition.
* `tasks/todo.md` for milestone execution tracking.
* `tasks/lessons.md` for reusable lessons and guardrails.
* `decisions/decision_log.md` for important decisions.
* `docs/project_context.md` for the durable project snapshot.

## Conditional product files

* `architecture.md` for current system structure when activated.
* `design/ui_spec.md` for project-specific UI rules when needed.
* `roadmap.md` for future phases only.

## Stub files activated later

* `contracts/` for data, API, and auth contracts.
* `tasks/test_plan.md`, `tasks/assumptions.md`, `tasks/risks.md` for deeper delivery control.
* `runbooks/` for debugging, deployment, and rollback guidance.
* `docs/integration_notes.md` for integration quirks.

## Design files

* `design/product_ui_system.md` for reusable premium UI guidance.
* `design/design_principles.md` for high-level design rules.
* `design/Apple-tier UIUX.md` and `design/UI_REDESIGN_SPEC.md` as reference material.
