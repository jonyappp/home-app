\# Project Brief



\## Problem



What problem does the product solve?



Keep this short.



\---



\## Target User



Who is the primary user?



\---



\## Core User Outcome



What should the user be able to accomplish?



One sentence.



\---



\## v1 Scope



Must be explainable in one sentence.



Example:



"User can track portfolio holdings across brokers."



\---



\## Must-Have Features



List only essential capabilities.



\---



\## Must-Not-Haves



Explicitly list features that are out of scope.



This prevents scope creep.



\---



\## Success Criteria



Define measurable outcomes.



Examples:



\- user can complete task in <30 seconds

\- dashboard loads under 2 seconds

\- system handles X data size



\---



\## Constraints



Technical or business constraints.



Examples:



\- API limitations

\- cost constraints

\- deployment restrictions
