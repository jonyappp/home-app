# Product UI System

## Design Philosophy

* Clean, minimal, high-signal UI.
* Premium feel without excess decoration.
* Reduce cognitive load before adding visual flourish.

## Layout Principles

* Use generous spacing to separate intent.
* Build clear hierarchy with layout first, styling second.
* Avoid clutter and competing focal points.

## Visual Language

* Use colour sparingly and deliberately.
* Prefer subtle surfaces and restrained contrast shifts.
* Avoid heavy gradients unless they serve hierarchy or feedback.

## Typography

* Establish a strong hierarchy between primary, secondary, and supporting content.
* Optimise for fast scanning and clear readability.
* Keep type scales consistent across views.

## Interaction Principles

* Provide responsive feedback for every meaningful action.
* Use smooth transitions that support understanding.
* Avoid unnecessary motion and decorative animation.

## Data Display

* Prioritise clarity over density.
* Use progressive disclosure for secondary detail.
* Present complex information in digestible layers.

## Anti-patterns

* Cluttered interfaces.
* Too many competing visual elements.
* Unnecessary complexity in navigation or controls.
